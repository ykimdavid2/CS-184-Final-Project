Summary:

In this project, we plan to compute an inverse kinematics model of a 2-degree-of-freedom robot leg. Additionally, we plan to model a single foot path of the robot leg as a set of continuous bezier curves. We will then interpolate through the foot path to obtain foot positions, which will then be passed through the inverse kinematics model in order to compute the angles of each joint in the robot leg, which we can then visualize in order to show how the robot leg moves through the foot path! There are many applications of topics we’ve learned about through class (inverse kinematics, interpolation, etc), and many variables to play with (limb lengths, interpolation methods, the bezier curve defining the foot path, etc…).


Problem Description:

In this project, we are attempting to build the computational foundation for the theory behind a robot dog. Making a stable, robust, and working robot dog is an active research problem, evidenced by many of the world’s premier research + robotics labs investing resources into it. As described in the summary, we are trying to demonstrate the theory behind modulating a robot dog pose through topics we’ve learned about in class (and a plethora of other skills that we’ll have to pick up along the way). It will be challenging to do this in the time allotted, since we have to span multiple layers of abstraction in our journey. For example, we will have to figure out the mathematical model behind a robot leg gait, whilst at the same time having to figure out how to animate this and the right platform to use to visualize our results via a programmatic interface. Just as in research, at every step along the way, there will be multiple decisions and variables that we can change, and we will have to maneuver bringing all of them in convergence to make a working animation of a mathematically-sound model of a robot log gait.


Goals and Deliverables

What we plan to accomplish:

  1. A sound mathematical inverse-kinematics model of a 2d robot leg (2 joints)
    • Given an xy position, the model should either return (1) the angles that the robot leg joints must achieve OR (2) if the robot leg cannot reach the desired xy position
  2. 2D environment to show the robot leg
    • Maybe a rudimentary jupyter notebook demonstrating the model, to allow us to try out inputs and see the outputs in a matplotlib plot
    • Determining what the best platform is to do the animation (Figuring out how to do this via openGL or something like it)
    • An environment where we can input joint angles, and see the output leg position
  3. Modeling the foot path as a set of bezier curves
  4. Sampling the foot path bezier curve via interpolation technique.
  5. An interactive environment where we can modulate limb lengths, foot path speed, key foot path parameters, etc and see the visual result.

What we hope to accomplish:

  1. Experimenting with different types of interpolations of the foot path.
  2. An investigation into how we can handle offsets and more complicated robot leg designs.
  3. A brief analysis of different ways to encode the foot path (is a set of bezier curves the right solution?).
  4. How to extend this into 3D?
  5. Can we use inverse kinematics on the body-level to figure out how to make the body twist and turn?
